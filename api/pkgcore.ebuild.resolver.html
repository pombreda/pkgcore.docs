Resolver that tries to minimize the number of changes while installing.

generate and configure a resolver that is focused on just installing requests- installs highest version it can build a solution for, but tries to avoid building anything not needed

Parameters:
  • vdbs – list of L{pkgcore.repository.prototype.tree} instances that represents the livefs
  • dbs – list of L{pkgcore.repository.prototype.tree} instances representing sources of pkgs
  • verify_vdb – should we stop resolving once we hit the vdb, or do full resolution?
  • force_vdb_virtuals – old style portage virtuals (non metapkgs) cannot be technically sorted since their versions are from multiple packages bleeding through- results make no sense essentially. You want this option enabled if you’re dealing in old style virtuals.
