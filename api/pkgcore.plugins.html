pkgcore.plugins.extend_path(path, name)[source]¶

Simpler version of the stdlib’s L{pkgutil.extend_path}.

It does not support “.pkg” files, and it does not require an __init__.py (this is important: we want only one thing (pkgcore itself) to install the __init__.py to avoid name clashes).

It also modifies the “path” list in place (and returns C{None}) instead of copying it and returning the modified copy.
